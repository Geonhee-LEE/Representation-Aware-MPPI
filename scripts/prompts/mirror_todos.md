# Mirror Notion TODO DB → repo-root TODO.md

You are running as a **non-interactive script** (typically cron or manual). Your job is a single one-shot: read the Notion TODO data source and write a deterministic Markdown snapshot to `TODO.md` at repo root so the user can grep/diff TODOs offline (e.g., on a flight, or while reviewing a PR diff).

Be terse. No Telegram. No Daily Log update. No commits. Just regenerate the file if its content would change.

## Setup
- Project root: `/home/geonhee/Representation-Aware-MPPI`. `cd` there.
- Notion TODO data source: `collection://b3ee1d96-a1f8-4d9e-98cf-6e9c4ef35239`.
- Today's date (KST): `TZ=Asia/Seoul date +%Y-%m-%d`.

## Steps

### 1. Fetch the TODO data source schema
Call `mcp__claude_ai_Notion__notion-fetch` on `collection://b3ee1d96-a1f8-4d9e-98cf-6e9c4ef35239` so you have the exact property names. Trust these (also documented in `auto_research.md`):
- `Title` (title), `Priority` (P0–P3), `Phase` (P0–P6), `Status` (Backlog/Today/Doing/Blocked/Done), `Owner` (claude/user), `Branch` (text), `NeedsUserTest` (checkbox), `Updated` (last_edited_time).

### 2. List all TODO pages
Call `mcp__claude_ai_Notion__notion-search` with:
- `data_source_url`: `collection://b3ee1d96-a1f8-4d9e-98cf-6e9c4ef35239`
- `query`: `TODO` (semantic — broad enough)
- `page_size`: 25 (max)
- `max_highlight_length`: 0
- `filters`: `{}`

If there are clearly more than 25 items, do a second call with a different query (e.g. `claude` then `user`) and merge by page id. Don't loop forever — 50 items total is plenty for a personal project.

For each result, fetch the page via `notion-fetch` to read full properties (search results omit them). Parallelize fetches up to 6 at a time.

### 3. Build the table
Group rows by `Status` in this order: `Doing`, `Today`, `Blocked`, `Backlog`, `Done`. Inside each group, sort by:
1. `Priority` ascending (P0 → P3, NULL last)
2. `Phase` ascending (P0 → P6, NULL last)
3. `Updated` descending

Cap `Done` to the most-recent **15** to keep the file scannable.

Each row renders as:

| Status | Priority | Phase | Owner | Title | Branch | UserTest |
|---|---|---|---|---|---|---|
| Doing | P0 | P0 | claude | … | autoresearch/… | ☐ |

- Title: strip surrounding `\[`/`\]` escapes. Truncate at 100 chars (append `…`). Replace `|` with `\|`.
- Branch: empty cell if blank; render the literal string otherwise (e.g. `main (3c6fc42)`).
- UserTest: `☑` if `NeedsUserTest=__YES__`, else `☐`.

### 4. Compose the file
Top of file:

```
# TODO (auto-mirror of Notion DB)

_Auto-generated by `scripts/mirror_todos.sh` — do not edit by hand._
_Source: Notion TODO data source `b3ee1d96-a1f8-4d9e-98cf-6e9c4ef35239`._

- Last update: `<KST timestamp>`
- Total open (Doing + Today + Backlog + Blocked): **<N>**
- Done shown: most recent <K> (full history in Notion)
```

Then per group: `## <Status>` heading, the table (or `_(none)_` if empty).

End with one line:

```
> Mirror generated <KST timestamp>. Re-run: `./scripts/mirror_todos.sh`.
```

### 5. Write only if changed
Read existing `TODO.md` (if present) via `Read`. If the new content equals the old (ignoring the two timestamp lines), exit without writing — keeps git noise down.

If the content differs, write the file via `Write`.

### 6. Stdout summary
One line:

```
MIRROR_DONE total=<N> doing=<n> today=<n> blocked=<n> backlog=<n> done=<n> changed=<true|false>
```

## Hard rules

- No commits, no push, no Telegram, no Daily Log update — this script is a pure file regenerator.
- Never write outside the repo. Only target `<repo>/TODO.md`.
- On any unrecoverable error: print `MIRROR_FAIL reason=<short>` to stdout and exit non-zero. Don't leave a half-written file.
