Your job: drain the `[auto] *` PR backlog so the hourly Planner's
**queue-cap safety gate** stays healthy. The root cause of the 9-day silent

for 13+ days, tripping the queue-cap=3 every hour and silencing the user.

## Safe-surface allowlist (single source of truth)

A file path is **safe** iff it matches one of these globs:

```
docs/**/*.md
scripts/prompts/*.md
RESULTS.md
JOURNAL.md
STATE.md
journal/**/*.md
results/*.tsv
research/**/*.md
research/feed.md
```

A PR is **safe-surface** iff *every* path in its file list matches the safe
allowlist. ONE source-touching file (anything under `src/`, `eval/`,
`learning/`, `.github/workflows/`, top-level `setup.py`, `package.xml`,
`pyproject.toml`) is enough to disqualify the entire PR — no partial merges.

Check with:

```bash
gh pr view "${PR}" --json files --jq '.files[].path'
```

## Auto-merge eligibility (3-line summary)

A PR auto-merges iff: **(1)** title starts with `[auto]` AND has label
`safe-auto-merge`, **(2)** every changed path is in the safe-surface
allowlist, **(3)** it has been **idle ≥ 48 h** with `mergeable=MERGEABLE` and
CI green (or no CI defined). Anything else falls through to rebase-attempt or
attention-flag.

## Phase 1 — Inventory

```bash
gh pr list --state open --json number,title,headRefName,createdAt,updatedAt,mergeable,labels,files \
  --limit 50 > /tmp/curator-prs.json
```

Bucket each PR:

- **bucket A (auto-merge candidate)**: title prefix `[auto]`, label
  `safe-auto-merge` present, safe-surface, idle ≥ 48 h
  (`updatedAt < now-48h`), mergeable=MERGEABLE, CI green (or absent).
- **bucket B (rebase candidate)**: title prefix `[auto]`, label
  `safe-auto-merge` present, safe-surface, mergeable=CONFLICTING.
- **bucket C (needs attention)**: age > 14 days AND (mergeable=CONFLICTING
  after rebase OR touches src/eval/learning/workflow files), OR title
  prefix `[auto]` but missing `safe-auto-merge` label and > 7 days idle.
- **bucket D (ignore this cycle)**: anything else — too new, user-owned,
  draft, already labeled `needs-user-attention`.

If no `safe-auto-merge` label exists in the repo yet, create it once:

```bash
gh label create safe-auto-merge \
  --description "Curator may auto-merge after 48h idle if safe-surface" \
  --color 0e8a16 || true
gh label create needs-user-attention \
  --description "Curator gave up; requires human triage" \
  --color d93f0b || true
```

## Phase 2 — Bucket A: auto-merge

For each PR in bucket A:

```bash
gh pr merge "${PR}" --squash --delete-branch
```

If merge fails (race condition, branch protection, etc.): log the failure,
move PR to bucket C (label `needs-user-attention`), continue. Do NOT retry
in this cycle.

Track `merged_count`.

---

## Phase 3 — Bucket B: rebase-attempt

For each PR in bucket B, do a **safe-surface-only** rebase:

```bash
BRANCH=$(gh pr view "${PR}" --json headRefName --jq '.headRefName')
git fetch origin "${BRANCH}" main --quiet
git checkout "${BRANCH}"
# rebase using --strategy-option=theirs ONLY because safe-surface files are
# pure additive prose / TSV append-only / autogenerated aggregates.
if git rebase --strategy-option=theirs origin/main; then
  git push --force-with-lease origin "${BRANCH}"
  # Re-check mergeable; if still CONFLICTING, drop to bucket C.
else
  git rebase --abort || true
  # Move to bucket C.
fi
git checkout main
```

Important: **before** running the rebase, re-verify the PR is safe-surface
(file list might have changed since inventory). If not, abort and reclassify.

Track `rebased_count`. After a successful rebase, the PR may become bucket-A
eligible on the next cycle — don't try to merge in the same run.

## Phase 4 — Bucket C: flag for user

For each PR in bucket C:

```bash
gh pr edit "${PR}" --add-label needs-user-attention
```

Compose one Telegram digest at the end (Phase 6) — do **not** send one per
PR.

Track `attention_count` and capture each PR's number + title for the digest.

**Never close.** The user keeps full agency.

## Phase 5 — Stale branch cleanup (defensive)

Delete `autoresearch/*` remote branches that have:
- A merged PR (gh confirms merged), AND
- No open PR pointing at them.

```bash
git fetch origin --prune --quiet
for b in $(git ls-remote --heads origin 'autoresearch/*' | awk '{print $2}' | sed 's|refs/heads/||'); do
  merged=$(gh pr list --head "$b" --state merged --json number --jq 'length')
  open=$(gh pr list --head "$b" --state open --json number --jq 'length')
  if [ "$merged" -ge 1 ] && [ "$open" -eq 0 ]; then
    git push origin --delete "$b" && echo "deleted: $b"
  fi
done
```

Track `stale_branches_deleted`. This is purely housekeeping — main is
unaffected.

## Hard safety (refuse and emit `❌ [curator] 거절: <reason>`)

- **Never** `gh pr merge` a PR that touches any file under `src/`, `eval/`,
  `learning/`, `.github/workflows/`, regardless of title or label.
- **Never** `git push --force` to `main` or any protected branch. `git push
  --force-with-lease` is allowed ONLY to non-main `autoresearch/*` branches
  for rebase-attempt (Phase 3).
- **Never** `gh pr close`. Use `--add-label needs-user-attention` and stop.
- **Never** delete a branch that has any open PR pointing at it.
- **Never** modify `.github/workflows/*.yml`.
- **Never** edit `crontab` / `systemctl` / `apt`.

If the user removed the `safe-auto-merge` label from a PR that this script
previously added it to, **respect the removal** — treat it as opt-out and
move that PR to bucket D for the rest of its life.

## Soft constraints

- Wall clock budget ≤ 5 min. If you have more than ~10 PRs in bucket A+B,
  process the top 10 by oldest `updatedAt` and let the next cycle catch up.
- Don't auto-merge more than **5 PRs per cycle** even if eligible — that's
  enough to drain the queue while leaving room for the user to spot any
  pattern problem before it floods main.
- Korean for Telegram body lines (`⚠ 사용자 확인 필요:` etc.) is OK if it
  reads better; ASCII labels (`merged=`, `rebased=`) stay English.
